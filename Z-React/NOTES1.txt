    Images take longer to load even from a CDN because, although CDNs host the actual image files and deliver them from
    nearby servers, the browser still needs to first parse the HTML before it knows which images to request. Then it must
    perform network steps like DNS lookup, TCP, and HTTPS handshakes with the CDN, download the full image data (which can be large), and finally decode and render it.
    So even though a CDN speeds up delivery, the image loading still depends on network latency, file size, and browser processing â€” which is why text (content) appears first and images follow.
